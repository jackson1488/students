# Файл: `backend/app/models/entities.py`

## Что это
Главный файл со схемой данных (таблицами) системы.

## Простыми словами
Каждый `class ... (db.Model)` в этом файле — это отдельная таблица в базе данных.

## Таблицы пользователей и учебной структуры
- `User` — учетные записи (логин, пароль, роль, группа, аватар).
- `StudyGroup` — учебные группы.
- `TeacherProfile` — ФИО, предметы, дата рождения, био преподавателя.
- `StudentProfile` — ФИО, дата рождения, био студента, ссылка на группу.
- `TeacherGroupBinding` — привязка «преподаватель ↔ группа ↔ предмет».

## Таблицы учебного процесса
- `Grade` — отдельные оценки.
- `Attendance` — посещаемость.
- `Homework` — задания, выданные группе или конкретному студенту.
- `HomeworkSubmission` — сдачи домашних заданий студентами.
- `ScheduleEntry` — расписание пар.
- `ScheduleTeacherLink` — связь пары и преподавателя.

## Таблицы тестирования
- `Test` — сам тест (предмет, модуль, таймер).
- `TestQuestion` — вопросы теста.
- `TestActivation` — кому и когда тест открыт.
- `TestAttempt` — попытка студента (ответы, балл, дедлайн).
- `ModuleScore` — устаревшая таблица модулей (для совместимости).
- `ModuleScoreEntry` — актуальные модульные баллы по предметам.

## Таблицы общения и контента
- `ChatMessage` — личный чат teacher ↔ student.
- `GroupChatMessage` — групповые чаты.
- `SupportMessage` — чат поддержки с админом.
- `Book` — локальные книги.
- `BookShelfItem` — личная книжная полка (избранное, закладки, прогресс чтения).
- `News` — новости и замены расписания.

## Что делает `to_dict()`
Почти в каждой модели есть `to_dict()`: он превращает запись БД в обычный JSON-словарь для API-ответа.
