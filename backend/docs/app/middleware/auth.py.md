# Файл: `backend/app/middleware/auth.py`

## Что это
Проверка авторизации и прав доступа.

## Простыми словами
Перед выполнением защищенного API этот файл проверяет:
1. Есть ли JWT токен,
2. Валиден ли токен,
3. Имеет ли пользователь нужную роль.

## Основные функции
- `token_required` — проверяет `Authorization: Bearer ...` и кладет пользователя в `g.current_user`.
- `roles_required(*allowed_roles)` — пускает только нужные роли (админ всегда имеет доступ).

## Какие ошибки возвращает
- `401` — токен отсутствует/битый/просрочен.
- `403` — роль не подходит.
