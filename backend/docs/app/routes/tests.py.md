# Файл: `backend/app/routes/tests.py`

## Что это
Самый крупный модуль: создание тестов, активация, прохождение, таймер, модульные баллы.

## Эндпоинты
### Управление тестом
- `POST /tests` — создать тест (teacher).
- `GET /tests/<test_id>` — детали теста (teacher/admin).
- `PUT /tests/<test_id>` — редактировать тест (teacher-владелец).

### Включение/выключение
- `POST /tests/activate` — активировать тест (admin).
- `POST /tests/deactivate` — досрочно закрыть тест (admin).
- `POST /tests/retake` — разрешить пересдачу конкретному студенту (admin).

### Для студента
- `GET /tests` — список доступных тестов.
- `POST /tests/start` — начать или продолжить попытку.
- `POST /tests/progress` — сохранить прогресс ответов.
- `POST /tests/submit` — завершить попытку и получить балл.

### Модули и итоговые баллы
- `GET /tests/module-summary/<student_ref>` — сводка модулей/экзамена/бонуса.
- `POST /tests/module-summary` — учитель выставляет модульные баллы.

## Простыми словами
- Teacher создает тест с вопросами и вариантами.
- Admin решает, кому и когда тест доступен.
- У студента одна попытка, отсчет времени хранится на сервере.
- Вопросы и варианты перемешиваются детерминированно (индивидуально для студента/активации).
- По завершению сразу считается результат.

## Важные механики
- Авто-завершение просроченных попыток.
- Блокировка редактирования, если кто-то уже проходит тест.
- Ограничения баллов модулей: модуль 1/2, экзамен, бонус.
