# Файл: `backend/app/routes/auth.py`

## Что это
Маршруты входа в систему и входа от имени другого пользователя (для админа).

## Эндпоинты
- `POST /login` — проверка логина/пароля, выдача JWT.
- `POST /auth/impersonate` — админ получает токен выбранного пользователя.

## Простыми словами
- Пользователь вводит логин и пароль.
- Backend сверяет пароль с хэшем.
- Если все верно, возвращает токен и данные профиля.
- Админ может быстро «переключиться» на студента/преподавателя без повторного логина.

## Особенности
- В ответе добавляется `full_name` (из профилей).
- Аватар отдается как полный публичный URL.
- Запрещено impersonate другого админа.
