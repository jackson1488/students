# Файл: `backend/app/__init__.py`

## Что это
Главный «сборщик» backend-приложения.

## Простыми словами
Этот файл:
- подключает все API-модули,
- создает таблицы,
- проверяет/добавляет новые колонки в старой БД,
- создает дефолтного админа,
- включает CORS,
- запускает фоновую задачу автозавершения просроченных тестов.

## Основные функции
- `register_blueprints(app)` — подключает все маршруты (`/login`, `/users`, `/tests` и т.д.).
- `seed_default_admin()` — гарантирует наличие `admin`.
- `normalize_legacy_roles()` — меняет старую роль `content_manager` на `scheduler`.
- `ensure_schema_updates()` — безопасные обновления схемы SQLite (добавление колонок/таблиц).
- `configure_cors(app)` — разрешает frontend обращаться к backend.
- `ensure_background_workers(app)` — запускает фонового воркера.
- `create_app()` — итоговая сборка приложения.

## Почему это важно
Если удалить `database.db`, именно этот файл помогает поднять рабочую структуру заново при первом запуске.
